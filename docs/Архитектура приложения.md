# Архитектура приложения

## Общее описание

Приложение реализовано по принципу клиент-серверной архитектуры и разделено на два основных слоя: фронтенд и бэкенд.   
Компоненты обмениваются данными через REST API с использованием JWT-токенов для аутентификации и авторизации.

Общая схема представлена ниже.

![image](Схема%20архитектуры.png)

## Frontend

Клиентская часть разработана с использованием **ReactJS**. 
В качестве менеджера состояния используется **MobX**, визуализации графиков реализованы через **React-ChartJs**,  
обработка и форматирование дат — через **date-fns**. В сборке используется **Vite**, стили написаны на **Sass**.

## Backend

Серверная часть построена на **Spring Framework (3.4.1)** и **Java 17**.  
Для работы с базой данных используется **jOOQ**, а миграции управляются с помощью **Liquibase**.  
Контейнерные интеграционные тесты реализованы с использованием **TestContainers**.  

Архитектура построена вокруг модели базы данных, которая включает в себя сущности:  
*	**user** – информация о пользователях ([скрипт](../src/main/resources/db/changelog/init/1_add_user_table.sql))
*	**category** – категории доходов и расходов ([скрипт](../src/main/resources/db/changelog/init/2_add_category_table.sql))
*	**transaction** – информация о транзакциях пользователей ([скрипт](../src/main/resources/db/changelog/init/3_add_transaction_table.sql))
*	**audit** – записи аудита операций ([скрипт](../src/main/resources/db/changelog/init/4_add_audit_table.sql), [пример заполнения таблицы](Аудит.md))

## Взаимодействие фронтенда и бэкенда

Компоненты приложения обмениваются данными через REST API. 
Аутентификация и авторизация реализованы через JWT (JSON Web Tokens), которые передаются в заголовках запросов.

## DevOps и развертывание

Для разработки и развертывания используется Docker и Docker Compose, 
что позволяет разворачивать как клиентскую, так и серверную часть приложения в изолированных контейнерах.

## Безопасность

Приложение использует JWT для безопасной авторизации пользователей и защиты маршрутов API.  
Также реализована валидация входных данных и обработка исключений на серверной стороне.
